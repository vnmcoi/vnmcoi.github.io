# Naive Approach (Graph Traversal)

To check whether vertex $u$ and vertex $v$ have the same connected component, a straighforward method is to use a graph traversal algorithm like Breadth-First Search (BFS) or Depth-First Search (DFS). For every type $2$ query, we can start a traversal from vertex $u$ and check if vertex $v$ is ever visited.

**Time complexity:** $\mathcal{O}(Q \times (V + E))$.

# Optimal Approach

Instead of traversing the entire graph every time we receive a query, we only need to care about the **relationship** between vertices. Specifically, we just need to know if two vertices share the same underlying connected component.

We can efficiently using the **Disjoint Set Union (DSU)** (also known as Union-Find) data structure. DSU allow us to dynamically track and merge connected component.

**Time complexity:** $\mathcal{O}(Q \times \alpha(N))$. Where $\alpha(N)$ is the inverse Ackermann function, which grows very slowly. In fact it grows so slowly, that it doesn't exceed $4$ for all reasonable $N$ (approximately $N < 10^{600}$).

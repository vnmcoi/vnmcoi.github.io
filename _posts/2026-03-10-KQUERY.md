# Lời giải sử dụng tìm kiếm nhị phân trên IT

Để trả lời truy vấn ta có thể tạo một cây IT trong đó mỗi node là một `vector` chứa các phần tử đã được sắp xếp tăng dần.

Dễ thấy kết quả truy vấn của `l r k` sẽ chính là số lượng số lớn hơn `k` trong các đoạn `[l; r]`. Để đếm nhanh ta có thể sử dụng tìm kiếm nhị phân.

**Lưu ý:** Để merge giá trị của hai node con lên cha thì ta sử dụng hàm `merge` để thực hiện duyệt và sắp xếp tăng dần hai vector.

**Tóm tắt nhanh về hàm merge:** Là một hàm được define sẵn bên trong C++ giống như bài toán hai con trỏ duyệt hai mảng để tạo thành một mảng duy nhất và mảng đó phải được sắp xếp tăng dần.

```cpp

#include <bits/stdc++.h>
using namespace std;

const int mxN = 1e5 + 5;

int N, Q;
int A[mxN];

struct SegmentTree
{
    int n;
    vector<vector<int>> it;

    SegmentTree(int _n) : n(_n)
    {
        it.assign((n << 2) + 1, {});
        build(1, 1, n);
    }

    void build(int id, int l, int r)
    {
        if (l == r)
        {
            it[id].push_back(A[l]);
            return;
        }

        int mid = (l + r) >> 1;

        build(id << 1, l, mid);
        build(id << 1 | 1, mid + 1, r);

        it[id].resize(r - l + 1);

        merge(it[id << 1].begin(), it[id << 1].end(), it[id << 1 | 1].begin(), it[id << 1 | 1].end(), it[id].begin());
    }

    int querry(int id, int l, int r, int a, int b, int v)
    {
        if (b < l || r < a)
        {
            return 0;
        }
        if (a <= l && r <= b)
        {
            return (it[id].end() - upper_bound(it[id].begin(), it[id].end(), v));
        }

        int mid = (l + r) >> 1;

        return (querry(id << 1, l, mid, a, b, v) + querry(id << 1 | 1, mid + 1, r, a, b, v));
    }
};

int main()
{
    ios::sync_with_stdio(0);
    cin.tie(0);

    cin >> N >> Q;
    for (int i = 1; i <= N; ++i)
    {
        cin >> A[i];
    }
    SegmentTree it(N);
    while (Q--)
    {
        int l, r, k;
        cin >> l >> r >> k;
        cout << it.querry(1, 1, N, l, r, k) << '\n';
    }
    return 0;
}

```

# Lời giải chuẩn

Có thể thấy rằng chúng ta hoàn toàn có thể tiết kiệm được chi phí tính toán các truy vấn vì nếu chúng ta trả lời được một truy vấn với `k = i` thì những truy vấn khác với `k = j` với `j < i` thì chắc chắn các số lớn hơn `i` cũng lớn hơn `j` nên ta không cần tính lại mỗi lần.

Vậy ta sẽ phát triển một thuật toán giải bằng cách sắp xếp tăng dần các truy vấn theo `k` rồi lần lượt trả lời offline từ `k` lớn nhất đến `k` nhỏ nhất.

Ở đây ta thấy nếu ta coi mảng `f` là một mảng giá trị chứa kết quả hay nếu như `A[i] > k[j]` (trong đó `k[j]` ở đây là truy vấn thứ `j` sau khi sắp xếp `k` tăng dần) thì tại vị trí `i` đó thì `f[i] = 1` ngược lại thì `f[i] = 0` hay dễ hiểu hơn ta thực hiện đánh dấu với những số `A[i]` nào lớn hơn `k` của truy vấn hiện tại thì ta bật giá trị là `1` trong mảng `f` và ngược lại thì ta đánh dấu là `0` khi đó thì ta có thể xử lý truy vấn nhanh bằng cách sử dụng IT hoặc BIT để trả lời truy vấn đoạn.

Vấn đề ở đây là nếu như mỗi lần chuyển truy vấn `k` mới thì ta phải duyệt lại mảng `A` một lần gây mất thời gian thì ta có thể sắp xếp `A` tăng dần rồi thực hiện duyệt truy vấn `k` và `A` sử dụng hai con trỏ khi đó mọi phần tử trong `A` tối đa chỉ bị duyệt qua một lần duy nhất và mỗi truy vấn `k` cũng chỉ được duyệt một lần duy nhất.

**Lưu ý:** Vì cả hai mảng `A` và truy vấn đều bị sắp xếp lại để trả lời offline nên chúng ta phải tiến hành lưu vị trí (thứ tự đầu vào) của cả các phần tử trong mảng `A` và truy vấn.
